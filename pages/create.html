Software Carpentry is an open project. Like all such projects, it needs your help to grow. If you see a mistake in on our website, please add a comment to the appropriate page or mail us and we'll fix it as quickly as we can.

Fork Us on GitHub

If you are interested in developing the Software Carpentry content, we have put our materials into these repositories on GitHub.

Repository Description Contributing Docs

v4 Companion repository to site holding the 4.0 teaching materials: online video lectures and slides. No longer under development

Pull requests are always welcome, or you can mail us to talk about what you'd like to do and how we can help.

New Lesson Material

We welcome new lessons and exercises (particularly lessons that show how the things we teach fit together). A good exercise has a single clear focus, and should be doable in five minutes or less; for examples, take a look at the "Challenges" sections of our introduction to the Unix shell.

For an example of what we'd like in a longer lesson, please look at the pull requests in our lesson repository. And if you have ideas of your own, please email us: we'd be happy to chat.

Related Projects

If you want to help us by programming, there are lots of things we need. Please get in touch if you'd like to tackle any of these, or have projects of your own you'd like to suggest.

Diffing and Merging
A groups of students have built a diff-and-merge tool for the IPython Notebook. Polishing this would help everyone in the IPython community, not just us.
Blocks and Images
ipythonblocks and skimage.novice have let us adopt a media-first approach to teaching programming. Enhancements to either of these are very welcome.
Browsercast
Gabriel Ivanica is building an HTML5-based tool for playing slideshows in sync with a voiceover. This project builds on earlier work; see this blog post for more information.
Peer Instruction on the Web
Piotr Banaszkiewicz is developing a web-based tool for peer instruction, a teaching technique which has been proven to be more effective than traditional lecturing. For more information, see this blog post or his GitHub repository.
Turtle Graphics
Turtle graphics goes all the way back to the original work at MIT on teaching kids to program and Paper's wonderful book Mindstorms. Several students at MacEwan University have built a prototype; further work would be very welcome.
A Perl Back End
The IPython Notebook doesn't have to use Python as the back-end process: they've defined a JSON-based protocol for communicating with arbitrary processes, and people have already built a replacement back-end in Julia (and are working on one in R). There are lots of Perl programmers in science; a Perl back-end would probably get a lot of love.
